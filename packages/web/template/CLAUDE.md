# Conjure web — codegen rules (frozen)

This document is the **immutable contract** for model-page codegen. The golden template is `src/pages/_template/` (an example `blog.Post` model). Every generated page is a clone/variation of this template; deviating from it is forbidden.

There are two delivery modes:

- **Runtime mode** — `src/pages/GenericModelPage.tsx` (list) + `src/pages/GenericModelDetail.tsx` (detail) render any model from the schema API at runtime: list, create, edit, delete, and inline children all work, no per-model code (`ManyToMany`/`JSON` are read-only — use codegen for bespoke controls). Install-and-go.
- **Codegen mode** — clone `src/pages/_template/` per model for full, owned, customizable pages. This document governs codegen mode.

## File structure (fixed 5 files)

```
src/pages/{model-kebab}/
  schema.ts    # zod form schema + Record type + model key constants + defaultFormValues
  columns.tsx  # ColumnDef[] — composed cell vocabulary only
  index.tsx    # list (FilterBar + DataTable + bulk + create button)
  form.tsx     # shared create/edit form (zod + RHF + server error → field mapping)
  detail.tsx   # /new | /:pk shared — form + meta card + InlineTable + ConfirmDeleteDialog
```

- **Read-only models** (schema `is_readonly: true`) are 3 files: `schema.ts` (Record type only), `columns.tsx`, `index.tsx` (no create/select/bulk) + a read-only detail (Dialog or form-less `<dl>` table). Prefer the Dialog approach for simple log models.
- Directory name is the model name in kebab-case (`BoardPost` → `board-post`, `SignalTalkComment` → `signal-talk-comment`).

## Absolute rules

1. API calls go through `@/lib/api`'s `adminApi` only. Direct fetch/axios is forbidden.
2. Query keys use the `queryKeys.*` helpers only: `[model, "list", params]` / `[model, "detail", pk]`.
3. Form types are derived from the zod schema with `z.infer` only. Hand-written interfaces are allowed for the Record type (`XxxRecord`) only.
4. **Required FK selection uses `superRefine`** (refine narrows the TS 5.5 output type to `number` and breaks the build — copy the `requiredFk` pattern from `_template/schema.ts`).
5. Cells use the composed vocabulary only: choices→`StatusBadge`, FK→`EntityLink`, currency→`MoneyCell`, plain number→`NumberCell` (right-aligned meta), delta/return→`DeltaCell` (finance convention: up=red / down=blue), datetime→`DateCell`, boolean→`BoolCell`, image→`ThumbCell`.
6. No hardcoded colors/dimensions outside the semantic tokens (`bg-white`, `text-[#...]`, `h-[32px]` are forbidden → `bg-surface`, `text-fg-muted`, `h-row`).
7. Do not modify `components/ui/`. Variations go in `components/composed/` as composition components and must be registered in `/style-guide`.
8. New pages are registered in `src/router.tsx`'s `modelRoutes` and `src/layouts/sidebar-nav.ts` (path convention `/m/{app_label.ModelName}`) — both are generated by `codegen/assemble.py`.
9. The compact variant is the default — use it without specifying `size`. `size="lg"` is forbidden.
10. Dates use `lib/format.ts` (display) and `lib/forms.ts` (input conversion) only. Fixed `yyyy-MM-dd HH:mm`.

## Field type → form control mapping

| Django field | zod | Control |
|---|---|---|
| CharField (no choices) | `z.string()` (+`.min(1, "...")` if required, `.max(n)` for max_length) | `Input` |
| CharField (choices) | `z.string()` or enum | `Select` — choices from the schema snapshot |
| TextField | `z.string()` | `Textarea` |
| Boolean | `z.boolean()` | `Checkbox` (label on the right) |
| Integer/Float/Decimal | `z.number().nullable()` (+superRefine if required) | `Input type="number"`; map ""→null via RHF `setValueAs` |
| ForeignKey/OneToOne | `requiredFk()` or `z.number().nullable()` | `FkCombobox` (initialLabel={record.{fk}_label}) |
| DateTimeField (editable) | `z.string().nullable()` | `Input type="datetime-local"` + `toDatetimeLocal`/`fromDatetimeLocal` |
| DateField | `z.string().nullable()` | `Input type="date"` + `fromDateInput` |
| Image/FileField | excluded from the form schema — manage with `useState<string \| File \| null>` | `ImageUploadField` + `mergeFilePayload` on submit (send only when File); show upload % via onProgress |
| JSONField | excluded (read-only `<pre>` display) | - |
| ManyToManyField | excluded from v1 form; show count only in detail | - (enhancement) |
| readonly field (`editable: false` or in readonly_fields) | excluded from the form | shown in the detail meta `<dl>` |

## Action patterns

Three kinds, distinguished by location:
1. **Page (toolbar) actions** — selection-independent, buttons in PageHeader `actions`. e.g. Excel export `<ExportButton model={MODEL} params={params} />` (reflects current filters/search, generic), create button.
2. **Bulk actions** — target selected rows. FilterBar `bulkActions` slot + `adminApi.bulk(model, {action:"update"|"delete", ids, data})`. e.g. the template's publish/unpublish/delete.
3. **Backend custom actions** — heavy server-side work (large streaming .xlsx, external sends, etc.). Add an endpoint to the api viewset, then call it from the front end. (The generic core has no action registry yet — enhancement.)

List pages include `<ExportButton>` in PageHeader `actions` by default.

## Data contract (guaranteed by the backend)

- Every record includes `_label` (str(obj)), `{fk}_label` for each FK, and `{field}_display` for each choices field — no extra lookups.
- List: `?page=&page_size=&ordering=&search=&{field}={value}&{field}__gte=` contract. Filters trust the schema's `list_filter` paths only.
- DRF validation errors are `{field: [messages]}` — copy the `setError` mapping pattern from `form.tsx`.
- Before delete, `/related/` warns about related objects → handled by `ConfirmDeleteDialog`.
- Inline children come from the schema `inlines` `{model, fk_field}` → `InlineTable`.

## UI copy (same wording across all screens)

- Create button/submit: "Add {singular noun}" / edit submit: "Save changes" / toasts: "Added." · "Changes saved." · "Deleted."
- Avoid abstract verbs like "Submit"/"OK". Errors state cause + fix ("Unit price must be 0 or greater").
- Empty list: "No {plural} yet. Add the first one." (read-only models close without a call to action.)
- Sentence case; consistent tone.

## Regeneration (when the schema changes)

- No blind overwrite — always diff-merge to preserve existing customizations.
- Files with customizations get a top-of-file `// @custom: <description>` comment to mark preservation priority.
- A git commit must precede regeneration.
